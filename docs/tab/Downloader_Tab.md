# Вкладка Downloader

Описание текущей реализации вкладки загрузчика: как собирается интерфейс, какие модули участвуют и как устроен поток загрузки/истории.

---

## Структура и сборка интерфейса

- Исходная верстка собирается из `templates/` в `src/index.html`.
- `src/js/modules/app/registerTabs.js` создает `#downloader-view`, а `renderDownloaderView` из `src/js/modules/views/downloaderView.js` оформляет содержимое вкладки.
- `renderer.js` остается точкой инициализации и не содержит UI-логики.
- Стили описаны в `src/scss/tab/_downloader.scss`, `src/scss/tab/_products.scss` и общих SCSS-компонентах.
- Основные элементы: строка ввода URL с иконкой источника, предпросмотр карточки, прогресс‑оверлей, группа кнопок (скачать, отмена, открыть папку, открыть последний файл), модал выбора качества, блоки очереди, история загрузок.

---

## Пользовательский поток

1. **Ввод ссылки** — `urlInputHandler.js` валидирует, показывает/скрывает кнопки, подтягивает предпросмотр (через IPC `get-video-info`), умеет добавлять весь плейлист в очередь кнопкой в превью.
2. **Автовставка** — `clipboardHandler.js` слушает буфер при фокусе окна, если ссылка поддерживается — переключает на вкладку, подставляет и триггерит предпросмотр.
3. **Выбор качества** — `downloadQualityModal.js` и `downloadQualityOptions.js` формируют варианты видео, видео без аудио, исходного аудио и MP3-конвертации.
4. **Старт/очередь** — `downloadManager.js` нормализует URL, защищает от дублей и передает задания в `downloadJobs.js`; лимит параллельности настраивается на 1 или 2.
5. **Процесс** — IPC `download-video` (main) запускает `download.js`: проверка/установка yt‑dlp/ffmpeg/ffprobe/deno, сбор информации, выбор форматов, качает/мерджит, трекает токен отмены. Прогресс (`onProgress`) красит прогресс‑бар и верхний индикатор (`downloadProgress.js`).
6. **Завершение** — `downloadManager.js` добавляет запись в историю (с кешированием превью через `cache-history-preview`), обновляет счетчик, сохраняет путь последнего файла (для кнопок "Последнее видео"/"Открыть папку").
7. **Отмена** — `downloadCancel.js` дергает IPC `stop-download`, сбрасывает стейты и прогресс.
8. **История** — `history.js` + `historyActions.js` + `filterAndSortHistory.js`: карточки, поиск/сортировка/очистка, предпросмотр превью в оверлее, отображение количества, всплывающие подсказки.

---

## Модули и связи

- **UI/поведение**: `downloadActions.js` (кнопки папки/последнего файла/старт), `downloadManager.js` (очередь, запуск, стейты), `downloadQualityModal.js` (парсинг форматов, выбор), `downloadProgress.js`, `downloadCancel.js`, `urlInputHandler.js`, `clipboardHandler.js`, `iconUpdater.js` (иконка источника).
- **Главный процесс**: `src/js/scripts/download.js` (управление инструментами, получение info, скачивание, мердж, отмена), IPC маршруты и состояние в `src/js/app/ipcHandlers.js` (download-video, stop-download, cache-history-preview и т.п.).
- **История**: публичный фасад `history.js`, реализация `features/history/`, а также `historyActions.js`, `historyFilter.js`, `filterAndSortHistory.js` и `normalizeEntry.js`.
- **Визуал**: `src/scss/tab/_downloader.scss` + общие компоненты (`components/_history.scss`, `components/_progress.scss`, `components/_inputs.scss` и т.д.).

## Ключевые особенности

- Поддержка многоссылочного ввода и очереди с защитой от дублей и трекингом количества.
- Динамическое определение качества: сортировка форматов по высоте/fps/битрейту, фильтрация по типу носителя, выделение "лучшая" кнопкой.
- Интеграция с историями: кеш превью, быстрый просмотр, поиск/сортировка, очистка, удаление выбранных.
- Прогресс с ETA на кнопке и оверлее, верхний мини‑индикатор, всплывающие тосты для всех состояний.
- Автовставка из буфера и предпросмотр плейлистов, кнопка "Добавить все" для массового добавления.
- IPC/главный процесс обеспечивает установку/обновление зависимостей (yt‑dlp/ffmpeg), отмену через токены, ограничение параллельных загрузок.

## Очередь загрузок

- **Добавление**:
  - Кнопка **«В очередь»** добавляет текущую ссылку в очередь без старта.
  - Shift+Enter в поле URL добавляет в очередь.
  - Плейлисты добавляются событием `queue:addMany`.
- **Запуск**:
  - Если уже идёт скачивание — новые задачи встают в очередь.
  - После завершения текущей загрузки берётся первый элемент из очереди и запускается.
  - Кнопка **«Запустить очередь»** позволяет стартовать первую задачу вручную, если сейчас ничего не качается.
  - Авто‑старт после обновления окна отключён: очередь ждёт ручного запуска.
- **Персистентность**:
- Очередь сохраняется в `localStorage`; состояние активных задач ведет `downloadJobs.js`.
  - При рестарте восстанавливается без автозапуска.
- **UI**:
  - Блок очереди показывает количество и иконки действий (**запуск**, **очистка**) с подсказками при наведении.
  - Добавлен список ссылок в очереди для наглядного просмотра (если кэш `get-video-info` доступен — показывается заголовок).
  - Каждый элемент очереди можно удалить отдельно кнопкой в списке.
  - Очистка сбрасывает очередь и обновляет UI.

## Проверка изменений

- При изменении шаблонов: `npm run templates:build`.
- При изменении SCSS: `npm run css:build`.
- При изменении логики: `npm run lint` и `npm test`.
- Актуальные пользовательские возможности описаны в `docs/APP.ru.md`; незавершенные идеи не следует хранить в этом документе как реализованные функции.
