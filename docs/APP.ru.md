# Руководство по приложению Thunder Load

Документ описывает текущие пользовательские возможности.

## Основные разделы

### Загрузчик

- Принимает введенную, вставленную или перетащенную ссылку и показывает превью.
- Предлагает видео, видео без аудио, исходную аудиодорожку и MP3.
- Добавляет отдельные ссылки и элементы плейлиста в сохраняемую очередь.
- Запускает одну или две параллельные загрузки, поддерживает отмену, открытие папки и последнего файла.
- Автовставка из буфера и автоматическое открытие выбора качества настраиваются.

Загрузчик использует управляемые `yt-dlp`, `ffmpeg`, `ffprobe` и Deno. Поддержка источников зависит от установленной версии `yt-dlp`; расширенные preview-функции ориентированы прежде всего на YouTube.

### Products

Раздел приводит товарные списки к единому формату:

- нормализует заголовки и названия;
- группирует позиции по секциям;
- формирует общую и дополнительную сводку;
- показывает сомнительные и некорректные строки;
- сравнивает результат с исходником;
- возвращает исправления во входной текст и копирует секции или весь результат;
- использует локальный редактируемый словарь правил.

### Инструменты

| Инструмент       | Назначение                                                            | Ограничения                                                       |
| ---------------- | --------------------------------------------------------------------- | ----------------------------------------------------------------- |
| WG Unlock        | Проверка и применение сценариев восстановления WireGuard              | Сценарии Windows и macOS отличаются                               |
| Проверка хеша    | Расчет хеша, сравнение файлов и история недавних результатов          | Кроссплатформенно                                                 |
| Media Inspector  | Контейнер, потоки, кодеки, bitrate, HDR и субтитры через `ffprobe`    | Требуется `ffprobe`                                               |
| Сортировщик      | Предпросмотр и сортировка файлов по расширениям и правилам конфликтов | Кроссплатформенно                                                 |
| Backup           | Профили резервного копирования, фильтры, логи и префлайт-проверки     | Использует системные средства архивации                           |
| Быстрые ярлыки   | Ярлыки питания, восстановления и системных утилит                     | Выполнение только на Windows; на macOS доступен developer-preview |
| WinGet Installer | Формирование, запуск и проверка WinGet-операций                       | Выполнение только на Windows; на macOS показывается preview       |

Часть инструментов можно скрыть в настройках. Backup находится внутри
`Инструментов`, а не в отдельной верхнеуровневой вкладке.

## История загрузок

История входит в раздел Загрузчика и поддерживает:

- поиск и фильтр по источнику;
- сортировку по дате, названию, размеру, качеству и источнику;
- пагинацию и настройку размера страницы;
- открытие файла, папки или страницы источника;
- повтор загрузки;
- удаление записи или локального файла;
- отмену поддерживаемых операций удаления;
- экспорт CSV и JSON.

## Настройки

В настройках доступны:

- язык, тема, размер шрифта и эффекты;
- стартовый раздел и видимость модулей;
- директория управляемых инструментов и поведение Загрузчика;
- профиль качества и лимит параллельных загрузок;
- поведение буфера обмена и окна;
- отображение статуса зависимостей;
- вид списка и лога Backup;
- доступность developer-инструментов.

Интерфейс приложения локализован на русский и английский языки.

## Данные и зависимости

Стандартные папки данных Electron:

- macOS: `~/Library/Application Support/Thunder Load`
- Windows: `%APPDATA%/Thunder Load`
- Linux: `~/.config/Thunder Load`

История, настройки, кеш превью, очередь и служебные данные находятся в профиле
приложения или renderer storage. Управляемые бинарники хранятся в выбранной
директории инструментов; их можно проверить, обновить, переустановить,
мигрировать или сбросить через менеджер.

## Поддержка платформ

Автоматические релизы содержат Windows NSIS и macOS DMG/ZIP для `x64` и `arm64`.
Эти сборки сейчас не подписаны. Linux AppImage можно собрать командой
`npm run build-linux`, но он не входит в автоматический release workflow.

На Linux `ffmpeg` и `ffprobe` могут потребовать установки через системный package
manager. Совместимые инструменты также могут находиться через `PATH`. Отдельные
функции имеют более узкие ограничения, указанные в таблице выше.
