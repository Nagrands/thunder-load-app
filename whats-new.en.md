# What's New

---

| What changed | What you get |
|---|---|
| The Products tab started the second diagnostics improvement pass | Diagnostics now add quick `All` / `Review` / `Typos` / `Duplicates` filters, and the normalized list can now be pushed back into the input in one step for manual cleanup and reruns |
| The Products tab received the first UX polish pass | The result is now marked as stale after source or dev dictionary edits, the dictionary now shows rule counts and malformed-line feedback, and the result area adds `Collapse all`, `Expand all`, and `Only uncertain` quick actions |
| Added the new Emerald color theme | The app now includes another calm dark option with emerald and teal accents, available in both Settings and the first-run flow |
| Developer mode now persists and controls more UI | It no longer resets after restart or language changes, can disable the Downloader tab, hides extra quick actions in the top bar and the footer while active, and no longer resurrects Downloader during tab fallback while correctly leaving an empty window if no visible tabs remain |
| Developer mode now includes a Products tab | The two-pane workspace now adds Paste, Clear, and Demo actions, a normalization diff, compact warnings for ambiguous lines, highlighted uncertain entries, an optional greens block, correct greenery bunch normalization in regular sections and in Store, cleaner result metrics split across separate rows, collapsible sections, normalization stats, rerun comparison, a dev replacement dictionary behind a separate button, restored internal scrolling in both columns, short `Summary` and `Greens` toggles with tooltips, lighter warning cards, a properly collapsible Normalization block, lighter per-section copy actions, a cleaned-up format action row, a wider dedicated layout for the Products workspace, instant result recalculation when `Summary` and `Greens` toggles change, comparison reset after service input replacement, a local dictionary dialog with proper focus handling, `Esc`, and backdrop close behavior, a more useful empty state with direct `Paste` / `Demo` actions, and clearer tooltip/feedback on per-section copy buttons |
