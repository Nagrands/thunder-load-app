# What's New

---

| What changed                                                                                | What you get                                                                                                                                                                                                                                                            |
| ------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Retrying from history opens quality selection again                                         | The “Retry download” action now uses the same preview and quality-selection flow as a pasted link, so you can choose the format before starting the repeat download                                                                                                      |
| Auto-opening quality selection no longer waits for ready formats                            | After a pasted link is recognized, the quality picker opens from the preview and title while formats keep loading inside the dialog, so new links no longer depend on a warm cache                                                                                       |
| History pagination is more compact                                                          | Page navigation and the page-size control now take less space under the list, avoid the footer, and keep the existing button and filter behavior                                                                                                                         |
| The Tips block was removed from Backup                                                      | Backup is more compact: the tips card, its rotation, and related background timers no longer appear or run                                                                                                                                                              |
| Backup profile delete confirmation no longer shows HTML tags                                | Profile names containing characters like `<b>` are shown as part of the name instead of raw HTML in the confirmation modal                                                                                                                                              |
| Scrollbars are now automatic                                                                | Scrollbars stay visually hidden by default and appear while scrolling, hovering a scrollable area, or focusing inside it                                                                                                                                                |
| Auto-opening quality selection after paste is more reliable                                 | The quality picker now reuses already loaded yt-dlp data without a second request, supports previews from the `thumbnails` list, and can reopen for an already recognized pasted link                                                                                   |
| Quality selection now copies file details                                                   | The old copy-URL action now copies the file title, channel, duration, and preview resolution; the title also has its own quick copy button                                                                                                                              |
| Quality selection now opens automatically after a recognized paste                          | When a pasted link resolves with a preview and formats, the quality picker opens by itself; the behavior can be disabled in Downloader settings                                                                                                                         |
| Queue start now lets you choose the launch mode                                             | When the queue has multiple items, Start asks whether to download only the first video or launch the whole queue; choosing the whole queue keeps the configured parallel download behavior                                                                              |
| Repeat downloads now check whether the file still exists on disk                            | If history still contains the URL but the file was removed, the download starts again and creates a new history entry; if the file still exists, Thunder Load warns that it is already downloaded and does not start a duplicate download                               |
| Hash Check now has a full file verification interface                                       | File input and settings are split into two working panels, the algorithm uses a Thunder Load dropdown, file size/readiness is visible, and the result panel updates with copy actions without reloading the view                                                        |
| The download quality dialog has a new compact design                                        | Format choices are easier to scan, the selected option is clearer, and source/preview actions now live in clean icon buttons while the existing download and queue flows stay the same                                                                                  |
| The setting that disables the Downloader tab is now hidden unless developer mode is enabled | The toggle no longer appears in regular mode, so the main tab cannot be hidden accidentally, while the developer-only option remains available only after explicitly enabling developer mode                                                                            |
| The “About app” section in Settings is now denser and more useful                           | It now presents the App/Electron/Chrome/Node versions in a tighter layout, while the “What’s new”, copy-info, and manual update-check actions take less space and stay available in one place                                                                           |
| Added an “About app” section to the Settings modal                                          | The settings sidebar now includes a dedicated entry with brief Thunder Load details plus the app and Electron versions, so the basic build information is available without checking the footer or system menus                                                         |
| Fully rebuilt the app brand mark and the macOS/Windows tray icons                           | Thunder Load now uses a single `T` brand mark instead of the old weather metaphor: the main icon is cleaner and more deliberate in the Dock and taskbar, while the macOS and Windows tray icons use a more compact circular `T` badge for better small-size readability |
| Removed app auto-close completely                                                           | The toggle and countdown no longer appear in Tools settings, the app no longer quits itself on the old delayed flow, and legacy settings for this feature are cleaned up on launch                                                                                      |
| Added background and live preview for YouTube links in Downloader                           | Pasting a YouTube URL still enables the muted decorative background preview, and now an explicit button can open a separate modal player with sound; when the preview source is refreshed, the player tries to continue the same video from roughly the same position   |
| Downloader tools status moved into the footer top-state                                     | The `yt-dlp`, `ffmpeg`, and `Deno` block now lives next to the current section label in the lower bar, appears only near the top of the page, and still respects the existing Settings toggle                                                                           |
| Section navigation now migrates from the header into the footer                             | Tabs stay in the header near the top of the page, then move into the lower bar while scrolling down together with a quick back-to-top action                                                                                                                            |
| Windows system quick shortcuts were updated in Tools                                        | Instead of Device Manager and Network Settings, the panel now creates shortcuts for `appwiz.cpl` (Programs and Features) and `cleanmgr` (Disk Cleanup)                                                                                                                  |
| Hash Check now has a compact redesign and fixed Drag and Drop on Windows                    | File selection and verification settings take less space, there is a quick reset action, and dropped Windows paths plus `file://` URIs are parsed correctly again                                                                                                       |
| File Sorter now uses a simpler layout, and the Russian locale labels it as «Сортировщик»    | Preview, sorting parameters, workspace controls, and tool settings are grouped into a more compact flow, so the working path is easier to scan and operate                                                                                                              |
