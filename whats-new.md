# Что нового?

---

| Что изменилось? | Что это даёт? |
|---|---|
| Форматтер списков лучше понимает шумный копипаст и контекстные названия товаров | Во вкладке «Продукты» строки с bullets, типографскими тире, кавычками, `NBSP`, `2x`/`2х`, скобками вокруг количества, контекстными alias вроде `репчатый лук`/`семрнко` и дробями вида `0, 500 гр` теперь нормализуются стабильнее, поэтому вставленный список реже требует ручной чистки |
| Во вкладке «Продукты» упрощена иерархия scroll-контейнеров | Промежуточные обёртки вокруг рабочей области теперь меньше режут overflow, поэтому прокрутка в исходном списке и результате должна реже упираться в вложенные контейнеры и вести себя предсказуемее |
| Стабилизирована высота вкладки «Продукты» при прокрутке | Форматтер списков теперь сильнее опирается на высоту родительского контейнера, а не на локальный `100vh`-расчёт, поэтому вертикальная прокрутка в исходном списке и результате должна вести себя ровнее и реже вызывать подвисания |
| В Formatter добавлено автообновление результата | Во вкладке «Продукты» теперь можно включить автоматический пересчёт: список заново форматируется сразу после изменений во входе, dev-словаре и переключателях, поэтому опытным пользователям не нужно вручную запускать форматирование после каждой правки |
| Исправлено зависание прокрутки после закрытия модалок и overlay | Блокировка скролла теперь управляется единым внутренним менеджером, поэтому после закрытия окон, how-to overlay и переходов между вкладками прокрутка корректно возвращается без перезагрузки страницы |
| Фон приложения стал легче и минимальнее | Основной фон теперь чище и спокойнее: он меньше отвлекает, сохраняет акцент активной темы и только слегка различается между экранами |
| Вкладка «Продукты» стала шире и выше на desktop | Форматтер списков теперь использует почти всю доступную ширину окна и получил больше рабочей высоты, поэтому исходный список и результат помещают больше контента без ощущения сжатости |
| Footer Downloader теперь показывает прямое действие для инструментов | Вместо постоянной кнопки «Переустановить» нижняя панель сама определяет состояние `yt-dlp`, `ffmpeg` и `Deno`: при отсутствии зависимостей предлагает `Установить`, при найденных обновлениях показывает `Обновить`, а подпись текущего раздела переименована во `Вкладку` |
| Исправлено фоновое preview YouTube во вкладке «Загрузчик» | Если `yt-dlp` возвращает playable Googlevideo-ссылку без явно заполненного контейнера, приложение теперь всё равно распознаёт формат и снова показывает приглушённое фоновое превью ролика |
| Для YouTube во вкладке «Загрузчик» добавлены video-preview и live preview | Под интерфейсом вкладки по-прежнему работает приглушённый фоновый preview, а по кнопке теперь можно открыть отдельное модальное окно с live player и звуком; при обновлении preview-source он старается продолжить и звук, и позицию того же ролика |
| Статус инструментов Downloader перенесён в footer верхнего состояния | Блок с `yt-dlp`, `ffmpeg` и `Deno` теперь живёт рядом с обозначением текущего раздела в нижней панели, показывается только у верхней границы страницы и по-прежнему отключается из Настроек |
| Навигация между вкладками теперь мигрирует из шапки в footer | В верхней части экрана вкладки остаются в header, а при прокрутке вниз автоматически переезжают в нижнюю панель вместе с кнопкой быстрого возврата наверх |
| Footer больше не отключается в режиме разработчика | Нижняя панель остаётся доступной даже при включённом developer mode, поэтому быстрые действия и статус раздела не пропадают |
| Контент вкладок больше не перекрывается footer | Верхнеуровневые экраны теперь резервируют место под нижнюю панель, поэтому рабочие области и нижние элементы не уходят под фиксированный footer |
| В «Инструментах» обновлены системные быстрые ярлыки Windows | Вместо «Диспетчера устройств» и «Параметров сети» теперь можно создать ярлыки «Установка и удаление программ» и «Чистка диска» |
| Панель быстрых ярлыков в «Инструментах» стала компактнее | Постоянный обзорный блок убран, верхняя часть секции занимает меньше места, а статус последнего действия появляется только после выполнения операции |
| «Проверка хеша» получила новый компактный интерфейс и исправленный Drag and Drop на Windows | Область выбора файла и параметры проверки теперь занимают меньше места, появился быстрый сброс состояния, а перетаскивание файлов с Windows-путями и `file://` URI снова корректно подхватывается |
| File Sorter в русской локализации переименован в «Сортировщик» и получил более простой интерфейс | Предпросмотр, параметры сортировки, настройки и рабочая область теперь сгруппированы компактнее, поэтому рабочий сценарий читается быстрее и требует меньше лишних переходов |
